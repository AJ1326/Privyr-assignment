Problem Statement

Create a simple web application to simulate a lead claiming process. The web app has 2 parts/pages:
1. ​Lead generator: ​a simple form where leads are generated. Leads have the following properties: Name*, Email, Phone Number, Comments. Only the name is compulsory
2. ​Lead Claiming portal for Agent A and Agent B:​ A simple page that shows leads claimed by the respective agents. Note these are two separate pages

The lead generation process should work like this:
- One can create leads using the form in 1
- Once a lead gets created in 1, it is available for claiming for agents A & B in portal 2.
- If a lead is claimed by Agent A, it cannot be claimed by Agent B (f​ astest finger first approach​). Vice-versa is true as well
- Once a lead has been claimed, it appears in portal 2 for that respective agent.
In order to avoid memery leak and considering design iteration, I have added it as add lead button at the top right corner.
